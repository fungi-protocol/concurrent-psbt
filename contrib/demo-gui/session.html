          Field editor

          Every field parses liberally (hex, base64, addresses — whatever the context admits). Unknown/proprietary raw keymap entries are editable hex rows (empty deletes the entry); decoded convenience fields validate locally but do not travel on save yet. Save runs the backend's save-time validation — violations come back with fix offers and explicit overrides — and mints a new fragment that replaces the edited one unless "keep" is checked.

          Assign unique ids

          Manual per-output ids are optional — the backend parses them liberally (hex/base58/bech32). Rows left blank are auto-assigned while auto-fill is on. Assigning mints a new fragment that replaces the source unless "keep" is checked.
